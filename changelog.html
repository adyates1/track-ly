Track-ly Change-log

Another way to prevent songs reaching your playlists!

Date: 10/6/2018, Version: 0.3.6

A feature that's been in the works for a while now has finally been implemented, you can now delete songs!

No more naughty songs!

Date: 9/6/2018, Version: 0.3.5

Don't want expletives at your party, we've got you covered. You can now say whether or not you're going to allow (what Spotify deem to be) explicit songs on your playlists.

Maximum number of songs can now be set

Date: 6/6/2018, Version: 0.3.4

Let's say your parties guest list is getting pretty out of hand, you're now allowed to set how many songs each user is allowed to add.

You can finally find out who's spamming the playlist with Nickelback

Date: 6/6/2018, Version: 0.3.3

Songs now display who's added them - this is pretty useful as you now know who to blame if your playlist starts getting filled with Nickelback garbage.

Settings have been added

Date: 6/6/2018, Version: 0.3.2

Now when users create a new room they're prompted with a set of settings, such as maximum songs and the like - hopefully this shall allow people to create playlists truly to their preferences.

We're back!

Date: 5/6/2018, Version: 0.3.1

Exams are over and we can now get back to working on Track-ly. We've implemented changes such as a character cap on names (after some very comical examples were sent to us). We also implemented some data sanitation for song queries. Also fixed some data issues, whereby lots of extraneous information was getting passed back to the user when they search for songs.

Beta release!

Date: 19/5/2018, Version: 0.3.0

After fixing up a few bugs we decided to release a beta version of our web app! It's gone down well so far, however there are a lot of apparent bugs. We've also had a lot of suggestions for new features and changes to the site.

Changed the name to Track-ly

Date: 7/5/2018, Version: 0.2.4

Unfortunately our original name, trackify, is already used in a .com domain name, we've decided to change to Track-ly.

Bug fixes galore and refresh tokens

Date: 6/5/2018, Version: 0.2.3

Over the past few days we've fixed a lot of small bugs, as well generated refresh tokens; which means that if the Track-ly's connection to your Spotify account is ever lost, then a new token is seamlessly generated without any supplementary user interaction.

User interface overhaul!

Date: 1/5/2018, Version: 0.2.2

The user interface was changed from a white theme to a much darker one using the bootstrap secondary colours, which makes Track-ly much easier on your eyes. Potentially in the future we may allow users to select their own style.

Live updates for playlists working

Date: 1/5/2018, Version: 0.2.1

If anyone has linked their Spotify account to Track-ly, any songs that are added through the web app will automatically be added the their Spotify playlist, as well as in-app.

Spotify playlists can now be made!

Date: 30/4/2018, Version: 0.2.0

Users can now sign into their Spotify account and have the playlist of songs queued added directly to their Spotify accounts!

Playlist generated for all users

Date: 29/4/2018, Version: 0.1.0

Users can now see all the songs in the playlist that have been added. Pretty much half of the functionality has been completed.

Searching for different things

Date: 28/4/2018, Version: 0.0.3

We got the requests to the API to now allow for searching for albums and artists as well as tracks. These required multiple calls to the API, one to get the ID for the artist or album in question and the other to return the list of songs relating to this.

Getting Spotify to play nice - the first of many all-nighters

Date: 27/4/2018, Version: 0.0.2

Bhav and I spent several hours getting the song searcher to work. Unlike what most (now outdated) documentation online says - the Spotify API now requires you to have a client ID and secret to even ask the API to return songs, this proved challenging as the AJAX request needed to be exceedingly specific just to get a list of songs, we were able to persevere and ended up getting it finished by around 1am.

Getting down to work

Date: 26/4/2018, Version: 0.0.1

The basic networking of the application was built on the first night, I (Alex) handled things such as creating new rooms, joining rooms and handling users disconnecting from rooms.

Initial Idea

Date: 25/4/2018, Version: 0.0.0

I (Alex) initially came up with the idea of an app which allows users to create playlist for parties and events and allow everyone at the events to automatically update to the playlist. I knew Spotify allowed users to create playlists and for people to update them. The issue however, is that you have to be either friends with them on Facebook/Spotify, find their playlist, and then they need to give you write access to the playlist (and so on and so forth for everyone else at the party). The basis of this app would be to remove these strenuous prerequisites and let users to get back to easily adding songs to the playlist and have them automatically added to any guest's playlists. I told this to Bhav at a bar and he seemed on board.
